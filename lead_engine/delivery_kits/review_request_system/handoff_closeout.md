## How to Send a Review Request

**If the owner sends manually:**

1. After the job is complete and the client seems satisfied, check the tracking record ([location]) to confirm you have not already sent a review request for this job.
2. If no prior request on file, open the approved template from the copy library (location: ______________________).
3. Copy the approved message and fill in [Client Name] and [service] with real details.
4. Send from your business phone (text) or email — not a personal number.
5. Log that you sent the request in the tracking record: client name, job, date sent.

**If automated:**
The review request sends automatically when the trigger condition is met. The owner does not need to take action — but the owner is responsible for confirming the client is satisfied before the trigger fires. If a job has a problem, pause the trigger before the send window passes (see Safe-Pause below).

---

## How to Pause or Disable (Safe-Pause Procedure)

If a review request should not go out — the client is unhappy, the job had an issue, or any other reason:

**For manual send:** Simply do not send the message. No tracking entry is made. No system change is needed.

**For automated send:**
1. Contact Drew immediately — before the send window closes.
2. Drew will pause or disable the trigger within [agreed response time, e.g., same business day].
3. Until Drew confirms the pause, log into [platform] and manually disable or skip the pending send.
4. Drew will confirm the pause in writing.

**Emergency contact:** [Drew's phone or email]

## The One-Per-Job Rule

One review request per completed job per client. No exceptions.

- Never send a second review request for the same job, even if the client didn't leave a review.
- Never send a review request to a client who expressed dissatisfaction. Handle the concern directly instead.
- The tracking record is how this rule is enforced — check it every time before sending.

---

## How to Handle a Negative Response

If a client responds to the review request with a complaint, dissatisfaction, or a negative comment:

1. Do not argue or debate.
2. Do not send a second review request.
3. Respond directly and professionally to address the concern.
4. Remove or do not activate any follow-up reminder for this client.

A negative response is not a reason to push harder — it is a signal to handle the relationship, not the review.

## Revision Boundary

**What is in scope without a change order:**
- Updating copy in an existing approved template (e.g., adjusting wording, correcting a phone number)
- Updating the review link if the GBP link changes
- Adjusting timing on the existing trigger

**What requires a change order:**
- Adding a new review destination (e.g., adding Facebook when only Google was set up)
- Adding a new trigger type
- Building automation where only manual send was originally configured
- Any mass-send or batch-send to past clients

Contact Drew to discuss before starting any out-of-scope change.
