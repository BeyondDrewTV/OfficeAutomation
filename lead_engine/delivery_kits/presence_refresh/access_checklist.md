# Presence Refresh — Access Checklist
Version: v1 | Kit: presence_refresh_v1

## Purpose
Confirm access to every platform in scope before starting any cleanup work. Missing access mid-job creates delays and broken half-states. Confirm upfront.

**Rule: Do not begin platform edits until all required access items for the active scope are checked.**

---

## Required Access (By Platform)

### Google Business Profile
- ☐ Owner has granted Drew manager access to the GBP listing
  - Path: GBP dashboard → Users → Add manager → Drew's Google account
- ☐ Access confirmed — Drew can see and edit the listing
- ☐ If the listing is unclaimed: owner will claim it first, then grant access

### Facebook Page (If In Scope)
- ☐ Owner has granted Drew editor or admin access to the Facebook page
  - Path: Page Settings → Page Roles → Add editor
- ☐ Access confirmed — Drew can see and edit the page
- ☐ If no Facebook page exists: confirm whether creating one is in scope (change order if not)

---

## Access Policy

Drew uses delegated access only. The owner is always the account owner. After the Presence Refresh is complete, Drew's manager/editor access can be removed — this is the recommended default unless a monthly support agreement is in place.

**Never request owner password or primary login credentials.**
