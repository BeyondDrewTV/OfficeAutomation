# Lead & Contact Setup — Access Checklist
Version: v1 | Kit: lead_contact_setup_v1

## Purpose
Confirm every required access point before starting any setup work. Missing access stops delivery. Catch it here, not mid-job.

**Rule: Do not begin configuration until all required items for the active scope are checked.**

---

## Required Access (By Active Scope)

### Phone / Missed-Call Path
- ☐ Access to phone number or call-forwarding settings
- ☐ Voicemail greeting access (if updating the greeting)
- ☐ Twilio account or messaging tool credentials (if auto-response is in scope)
- ☐ Test phone number confirmed for verification calls

### Form / Email Routing
- ☐ Access to website contact form destination settings (or owner will forward inbound)
- ☐ Access to the email inbox or alias receiving inquiries
- ☐ Ability to configure or test auto-reply (or owner confirms it is already set correctly)

### Lead Logging
- ☐ Google Sheet shared with Drew's Google account (or equivalent logging destination)
- ☐ Write / append permissions confirmed with a test row

### Owner Notification
- ☐ Owner has confirmed their notification phone number or email
- ☐ Test notification destination ready — send one test before go-live

---

## Optional (Only If In Active Scope)

- ☐ CRM write access (if lead log feeds an existing CRM)
- ☐ Facebook / Instagram notification setup (if social DM routing is included)

---

## Access Policy

After setup is complete, the owner retains all account access. Drew does not hold long-term access to phone systems, email inboxes, or messaging tools unless explicitly part of a signed monthly support agreement.

**Delegated access only. Owner is always the account holder.**
